## Execution Workflow Preferences

- **Review-First Workflow**: Antigravity must act as an execution and recommendation layer, not as a blind automation tool.
- For every major action, output a review-first workflow where the user can open the result, read the recommendation, inspect the generated draft or fix, and then explicitly choose to approve, edit, reject, or regenerate.
- The user's role is lightweight: they should not need to do technical work manually. Antigravity handles the technical execution and the user simply reviews the proposed changes and approves the next step.
- **Implementation Plan Requirements**: Every Implementation Plan must clearly show:
  1. Goal
  2. Exact files/components affected
  3. Risks
  4. Acceptance criteria
  5. Visible UI outcome
