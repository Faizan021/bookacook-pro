## Execution Workflow Preferences

- **Review-First Workflow**: Antigravity must act as an execution and recommendation layer, not as a blind automation tool.
- For every major action, output a review-first workflow where I can open the result, read the recommendation, inspect the generated draft or fix, and then explicitly choose to approve, edit, reject, or regenerate.
- My role should be lightweight: I should not need to do technical work manually. Antigravity handles the technical execution and I mainly review the proposed changes and approve the next step.

## Implementation Plan Requirements
Every Implementation Plan must clearly show:
1. Goal
2. Exact files/components affected
3. Risks
4. Acceptance criteria
5. Visible UI outcome
